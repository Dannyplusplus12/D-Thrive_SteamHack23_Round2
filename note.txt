Vào /create để post bài

/create có lưu trạng thái bài đang viết lần trước bằng Current_Post bên models
em để nó lưu được nhiều Current_Post để sau có thể nâng cấp lên kiểu.. lưu nhiều bài chưa hoàn thiện

Nếu render content từ thằng tiny ra html bằng Flask thì phải thêm thằng | safe
như vở file preview.html thì em có ghi:
{{ title | safe }}
{{ content | safe }}

# để xoá một post thì gởi request API tới /delete-post với nội dung là id của post muốn xoá
# em đã tạo sẵn một hàm deletePost ở file base.html
# khi bấm xoá post thì gọi hàm deletePost và truyền tham số là id của post

#em chưa test thử cái xoá nên có thể có bug, có gì anh báo em


# khi post bài ở preview.html thì nó sẽ gởi request với methods POST tới thằng /blog
# tức là thằng /blog mới là thằng thực hiện chức năng đang bài
# các lấy post hiện có thì em đã ghi trong thằng /blog ở views.py

# thằng tags ở mỗi post em đang lưu dưới dạng string
# nếu anh muốn render tags ra thì anh chỉ cần tách string theo dấu cách
# tức là trong một tag không có dấu cách, theo em thì cách đó nhanh gọn nhất

# thằng title với thằng tags là string bình thường
# còn thằng content là html string (ví dụ "<p>Hello world!</p>")
# nên anh muốn render thằng content thì anh phải thêm | safe như em nói ở trên

# để edit một post thì anh gọi đến /<int:id>/edit với <int:id> là id của post muốn edit
# một cách dễ là tạo 1 thẻ a với href="{{ url_for('views.edit', id=post['id']) }}
